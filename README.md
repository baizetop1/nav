## 标签扩展（Tagging）

本仓库已支持为站点添加标签，并在页面顶部显示“标签筛选栏”。

### 如何为站点添加标签

- 在 `config.yml` 中的站点条目里新增 `tags` 字段，示例：

```yaml
content:
  categories:
    - name: 常用网站
      sites:
        - name: 安全树洞
          description: 个人博客
          url: https://baizeone.top/
          tags: [博客, 安全]
```

### 工作流如何生成标签数据

- GitHub Actions 会在生成 `index.html` 后，解析 `config.yml` 中的 `tags` 字段，生成 `tags.json`，并在页面末尾自动注入 `tags.js`。
- `tags.js` 会在页面加载后创建标签工具条，点击标签即可筛选对应的站点卡片。

### 注意事项

- 未添加 `tags` 的站点仍会显示在“全部”视图中。
- 标签大小写敏感建议统一小写或使用中文，便于管理。
- 如需自定义样式，可修改根目录下的 `tags.js` 内联样式。

## 现代科技风主题（Modern Tech Theme）

本仓库已引入 `modern.css`，在不改动原有结构的基础上通过覆盖样式实现：

- 霓虹渐变背景与低饱和动效（可读性优先）
- 玻璃拟态卡片（透明度与模糊、低对比描边）
- 侧边栏与链接霓虹高亮、轻量交互动画

### 使用与自定义

- 样式文件：`modern.css`（根目录）
- 工作流会在生成后的 `index.html` 自动注入：
  - `<link rel="stylesheet" href="modern.css">`（插入到 `<head>`）
  - `<script src="tags.js"></script>`（插入到 `<body>` 末尾）
- 如需调整配色，可编辑 `modern.css` 的 `:root` 变量（如 `--accent`、`--accent-2` 等）。
- 若需要更强动效，可提升 `@keyframes bg-pan` 的位移或时间，但建议谨慎以避免晕动症。
